# Image Upload Guide — Meet the Whytes

Use this reference when preparing images before uploading to Sanity Studio.

---

## Key Rules

- **Sanity always delivers WebP** to browsers — upload format only affects storage, not what visitors see.
- **Upload format:** WebP (preferred, smallest storage) or JPEG. PNG only for transparent-background images (product shots, logos).
- **Export quality:** WebP at 80% or JPEG at 85% when exporting from your tool.
- **Delivered file size is NOT the upload size.** A 1200×800 upload is delivered at whatever size the page needs — a listing card receives ~80–150 KB, not the full source.
- **2× rule:** source width must be at least 2× the largest display size so images stay sharp on Retina/HDPI screens.

## Recommended Export Tools

| Tool | Notes |
|------|-------|
| **Squoosh** (squoosh.app) | Free, browser-based — best for resizing + WebP export |
| **Canva** | Export as JPEG at high quality; download "WebP" if available |
| **Lightroom / Photoshop** | Export as JPEG 85% or WebP 80% |
| **iPhone / Google Photos** | Download original before uploading; HEIC files should be converted to JPEG first |

---

## Common Mistakes to Avoid

- Uploading tiny screenshots or phone thumbnails — they will look blurry on desktop
- Using a 16:9 landscape image for product/essentials slots (they need a **square**)
- Uploading 850 × 405 images — these are too narrow for the new hero layouts (need 1200 px wide)
- Uploading HEIC files directly — convert to JPEG or WebP first
